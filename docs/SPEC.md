# 献立カレンダー 設計仕様書

## 概要

10日分の献立を書き込めるカレンダーアプリ。  
静的HTMLとして配信し、Windows Edge で常時タブ表示する。

---

## 画面構成

### テーブルレイアウト

```
┌──────────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
│今週の献立 │  朝  │      │  昼  │      │      │  夜  │      │      │
├──────────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│          │ パパ │ ぼく │ パパ │ ママ │ ぼく │ パパ │ ママ │ ぼく │
├──────────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│5月21日(木)│      │      │      │      │      │      │      │      │
│   ...    │      │      │      │      │      │      │      │      │
│5月30日(土)│      │      │      │      │      │      │      │      │
└──────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

- 列構成: 日付(1列) + 朝(2列) + 昼(3列) + 夜(3列) = 計9列
- 行構成: ヘッダー行(1行) + 人物名行(1行) + 日付行(10行) = 計12行

## 機能仕様

### セル編集

| 状態 | 表示 | 操作 |
|------|------|------|
| 通常 | テキスト表示 | ダブルクリックで編集モードへ |
| 編集中 | textarea 表示 | 編集中セル以外をクリックで確定 |

- **編集対象**: 日付セル以外のすべてのセル（ヘッダー・人物名・献立）
- **文字数制限**: 100文字まで
- **改行**: 可能（Shift+Enter または Enter）
- **確定トリガー**: `blur` イベント（フォーカスアウト）

> **ヘッダー・人物名セルの注意点**  
> 「今週の献立」「朝/昼/夜」「パパ/ぼく/ママ」は編集可能だが、
> 現在値のみ保存（日付ごとの履歴は持たない）。

### 日付表示

- 表示範囲: 今日から10日分（今日 〜 今日+9日）
- 並び順: 昇順（今日が最上行）
- フォーマット: `5月24日(日)` ※ゼロ埋めなし
- リロード時: 今日を基準に再計算して表示

### 色分け

| 条件 | 背景色 |
|------|--------|
| 土曜・日曜の行 | `#FFE4EC`（薄ピンク） |
| 平日の行 | なし（デフォルト） |

## データ設計

### ストレージ

- **方式**: `localStorage`
- **キー**: `weekly-meal-data`
- **保存形式**: JSON

### データ構造

```typescript
type AppData = {
  /** ヘッダー行テキスト（編集可能） */
  headers: {
    title: string;    // デフォルト: "今週の献立"
    morning: string;  // デフォルト: "朝"
    noon: string;     // デフォルト: "昼"
    night: string;    // デフォルト: "夜"
  };
  /** 人物名行テキスト（編集可能・現在値のみ保持） */
  persons: {
    morning: [string, string];        // デフォルト: ["パパ", "ぼく"]
    noon: [string, string, string];   // デフォルト: ["パパ", "ママ", "ぼく"]
    night: [string, string, string];  // デフォルト: ["パパ", "ママ", "ぼく"]
  };
  /** 献立データ（30日分） */
  meals: {
    [dateKey: string]: {              // キー形式: "2026-05-21"
      morning: [string, string];
      noon: [string, string, string];
      night: [string, string, string];
    };
  };
};
```

### 保存タイミング

- **トリガー**: セル編集確定時（`blur` イベント）
- **処理フロー**:
  1. React state を更新
  2. `localStorage` に JSON.stringify して即時書き込み
  3. 30日より古い `meals` エントリを削除（クリーンアップ）

### データ保持期間

- 献立データ: **30日分**（今日から過去30日）
- ヘッダー・人物名: 制限なし（現在値のみ）

## 状態管理

```
App (useState: AppData)
  │
  ├── useMealStorage カスタムフック
  │     ├── 初期化: localStorage から読み込み → なければデフォルト値
  │     ├── save(data): localStorage に書き込み + 古いデータ削除
  │     └── getDefault(): デフォルト AppData を返す
  │
  └── MealCalendar (props: AppData + onCellChange)
        ├── CalendarHeader        ← ヘッダー行
        ├── PersonHeader          ← 人物名行
        └── DayRow × 10          ← 日付行
              ├── DateCell        ← 読み取り専用
              └── EditableCell × 8
```

### セル変更の流れ

```
EditableCell (blur)
  → onCellChange(path, value) コールバック
    → App: state 更新
      → useMealStorage.save(newData)
        → localStorage に書き込み
```

## コンポーネント仕様

### `EditableCell`

| props | 型 | 説明 |
|-------|----|------|
| `value` | `string` | 表示・編集する文字列 |
| `onChange` | `(value: string) => void` | 確定時コールバック |
| `maxLength` | `number` | 最大文字数（デフォルト100） |

**動作**:
- 通常時: `<div>` としてテキスト表示（空の場合は空白を確保）
- ダブルクリック時: `<textarea>` に切り替え、フォーカス移動
- `blur` 時: `onChange` を呼び出し、`<div>` に戻る

### `DayRow`

| props | 型 | 説明 |
|-------|----|------|
| `date` | `Date` | 対象日付 |
| `meals` | `DayMeals \| undefined` | 当日の献立データ |
| `onCellChange` | `(col: number, value: string) => void` | セル変更コールバック |
| `isWeekend` | `boolean` | 土日フラグ（背景色制御） |
